---
title: "Sucursales"
description: "Manage branch office locations used in trip creation and collaborator assignments."
---

Sucursales are the branch offices that employees travel to. Each sucursal has a name, an address, and a status. Branches are referenced by both the Colaboradores and Viajes modules.

## Data model

| Column | Type | Description |
|---|---|---|
| `sc_id` | `INT` (auto PK) | Auto-generated primary key |
| `sc_nombre` | `VARCHAR` | Branch name |
| `sc_direccion` | `VARCHAR` | Branch address |
| `sc_estado` | `TINYINT` | `1` = active, `0` = soft-deleted |

## Branch name format

Branch names follow a `{City Code} - {Neighborhood}` convention:

| Example name | Address |
|---|---|
| `SPS - Armenta` | Boulevard de Armenta |
| `TGU - San ignasio` | 4ta Calle Residencial San Ignacio |
| `CB - Villa Neen` | 5ta Calle 3ave |

<Tip>
Use a consistent city code prefix (e.g., `SPS` for San Pedro Sula, `TGU` for Tegucigalpa, `CB` for ComayagĂĽela) so that branches are easy to identify in trip and collaborator dropdowns.
</Tip>

## CRUD operations

### List branches

```
GET /sucursales
```

Renders the sucursales management page. All branches are returned regardless of `sc_estado`, allowing administrators to see both active and soft-deleted records.

### Create a branch

```
POST /sucursales/create
```

<ParamField body="sc_nombre" type="string" required>
  The name of the branch office. Use the `{City Code} - {Neighborhood}` format.
</ParamField>

<ParamField body="sc_direccion" type="string" required>
  The physical address of the branch office.
</ParamField>

### Update a branch

Fetch the current branch data for the edit form:

```
GET /sucursales/update/:id
```

Submit the updated data:

```
POST /sucursales/update
```

The same fields apply as for creation (`sc_nombre`, `sc_direccion`).

### Soft delete

```
POST /sucursales/delete/:id
```

Sets `sc_estado = 0` on the branch record. The branch is not permanently removed from the database.

<Warning>
Soft-deleting a branch removes it from the active dropdown in the Viajes form. Existing trips that reference the branch are not affected.
</Warning>

## Effect on other modules

<Columns cols={2}>
  <Card title="Viajes" icon="car" href="/features/viajes">
    Only sucursales with `sc_estado = 1` are loaded into the trip creation form.
  </Card>
  <Card title="Colaboradores" icon="users" href="/features/colaboradores">
    Branches are selectable when registering a collaborator and assigning a distance.
  </Card>
</Columns>

| Method | Route | Description |
|---|---|---|
| `GET` | `/sucursales` | List all branch offices |
| `POST` | `/sucursales/create` | Create a new branch |
| `GET` | `/sucursales/update/:id` | Fetch branch data for the edit form |
| `POST` | `/sucursales/update` | Update a branch record |

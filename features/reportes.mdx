The Reportes module provides a single, consolidated view of all recorded trips. It joins the `viajes`, `usuarios`, `sucursales`, and `transportistas` tables to display a human-readable report with everything an administrator needs to review travel activity.

## Accessing the report

```
GET /reportes
```

This route is available to authenticated users. It executes a multi-table JOIN query and renders the results on the reports page.

## Report columns

| Column | Source table | Field | Description |
|---|---|---|---|
| Employee name | `usuarios` | `us_nombre` | The name of the collaborator who made the trip |
| Branch | `sucursales` | `sc_nombre` | The branch office visited |
| Carrier | `transportistas` | `tr_nombre` | The carrier used for transport |
| Cost | `viajes` | `vj_costo` | The recorded cost of the trip |
| Distance | `viajes` | `vj_recorrido` | Route distance in km |
| Date | `viajes` | `vj_fecha` | The date the trip took place |

## Underlying query

The report is generated by a raw SQL query that joins all four tables:

```sql
SELECT us.us_nombre,
       suc.sc_nombre,
       tr.tr_nombre,
       vj.vj_costo,
       vj.vj_recorrido,
       DATE(vj.vj_fecha) AS vj_fecha
FROM viajes AS vj
INNER JOIN usuarios AS us ON us.us_id = vj.us_id
INNER JOIN sucursales AS suc ON suc.sc_id = vj.sc_id
INNER JOIN transportistas AS tr ON tr.tr_id = vj.tr_id
```

### Join breakdown

| Join | Condition | Purpose |
|---|---|---|
| `viajes â†’ usuarios` | `us.us_id = vj.us_id` | Resolves the employee name from the user ID stored on the trip |
| `viajes â†’ sucursales` | `suc.sc_id = vj.sc_id` | Resolves the branch name from the branch ID stored on the trip |
| `viajes â†’ transportistas` | `tr.tr_id = vj.tr_id` | Resolves the carrier name from the carrier ID stored on the trip |

<Note>
The query uses `DATE(vj.vj_fecha)` to strip the time component from the timestamp, returning only the date portion for display.
</Note>

## Practical use cases

<AccordionGroup>
  <Accordion title="Reviewing total travel costs">
    The report lists `vj_costo` for every trip. Administrators can scan or export this data to sum up transportation expenses per employee, branch, or carrier.
  </Accordion>
  <Accordion title="Auditing trip distances">
    Since the system enforces a maximum of 150 km per trip at creation time, the report provides a record of all approved route distances. Any value in the `vj_recorrido` column will be between 1 and 150.
  </Accordion>
  <Accordion title="Tracking carrier activity">
    The `tr_nombre` column shows which carrier was used for each trip. This helps identify the most frequently used carriers or compare costs across providers.
  </Accordion>
  <Accordion title="Branch travel frequency">
    The `sc_nombre` column shows which branch each trip was destined for. This can highlight which locations generate the most employee travel.
  </Accordion>
</AccordionGroup>
